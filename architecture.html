Game Loop

Every animation frame, the loop fires in PacmanGame.tsx via requestAnimationFrame. Delta-time is capped at 50 ms to prevent physics tunnelling on slow frames. All game state lives in refs — not React state — to keep the closure hot without triggering re-renders.

Browser fires each display frame. Computes dt = min(Δt, 50ms) to avoid big jumps on slow frames. Accumulates globalTimer to throttle React HUD re-renders to ~100 ms intervals.

05

Ghost AI — Four Personalities

At every intersection, a ghost picks the direction that minimises Euclidean distance to its current target tile. Reversing into the previously occupied tile is forbidden (except on mode transitions). Each ghost exits the ghost house after a staggered delay.

👾
BLINKY — Red
Chase: Targets Pac-Man's exact current tile. Direct pursuit, no prediction — pure aggression. Fastest reaction to position changes.

Scatter: Top-right corner (tile 25, 0).
Speed: 2.0 px/frame · Exit delay: 0 ms (exits first)
🎀
PINKY — Pink
Chase: Targets 4 tiles ahead of Pac-Man's current direction — attempts to cut off escape routes and set up pincer attacks with Blinky.

Scatter: Top-left corner (tile 2, 0).
Speed: 2.0 px/frame · Exit delay: 2 000 ms
💠
INKY — Cyan
Chase: Complex pivot — takes 2 tiles ahead of Pac-Man, draws a vector from Blinky to that point, then doubles it. Makes Inky highly unpredictable and dependent on Blinky's position.

Scatter: Bottom-right corner (tile 27, 30).
Speed: 2.0 px/frame · Exit delay: 5 000 ms
🟠
CLYDE — Orange
Chase: Chases Pac-Man directly when distance² > 64. Switches to his own scatter corner when closer than 8 tiles — appearing to wander away aimlessly when near.

Scatter: Bottom-left corner (tile 0, 30).
Speed: 2.0 px/frame · Exit delay: 8 000 ms (exits last)
Frightened & Eaten Modes

Power pellet → all ghosts enter FRIGHTENED (8 s, speed 1.2 px/frame, random direction at intersections, reverse immediately on activation). Colliding with a frightened ghost → EATEN (speed 4.0 px/frame, eyes only, targets own home pixel to respawn). Ghost scores multiply per chain: 200 → 400 → 800 → 1 600. Frightened ghosts flash as a warning during the final 2 seconds.

Code Peek — Key Snippets

Three patterns that define how the codebase works: the RAF loop, the ghost targeting switch, and the immutable state spread.

RAF Loop · PacmanGame.tsx
const loop = (timestamp: number) => {
  const dt = Math.min(timestamp - lastTimeRef.current, 50); // cap at 50ms
  lastTimeRef.current = timestamp;

  updatePowerPulse(dt); // animate pellet glow globally

  if (gameData.state === 'PLAYING') {
    const newState = updateEngine(state, dt, inputDirRef.current);
    stateRef.current = newState;
    setUiState({ score, highScore, lives, level, gameState });
  }

  ctx.fillStyle = '#000010';
  ctx.fillRect(0, 0, CANVAS_W, CANVAS_H);
  renderMaze(ctx, s.gameData.maze);
  renderParticles(ctx, updatedParticles);
  renderGhost(ctx, ghost, globalTimer);
  renderPacman(ctx, s.player, s.dyingFrame);

  animFrameRef.current = requestAnimationFrame(loop);
};
Ghost Target Selection · ghost.ts
function getTarget(ghost, player, allGhosts, maze): Vec2 {
  if (ghost.mode === 'EATEN')      return GHOST_HOME_PIXEL[ghost.name];
  if (ghost.mode === 'FRIGHTENED') return { x: Math.random() * MAZE_COLS, y: ... };
  if (ghost.mode === 'SCATTER')    return tileToPixel(ghost.scatterTarget);

  // CHASE — personality routing
  switch (ghost.name) {
    case 'BLINKY': return tileToPixel(player.tilePos);
    case 'PINKY':  return tileToPixel({ x: px + dv.x * 4, y: py + dv.y * 4 });
    case 'INKY': {
      const ahead = { x: px + dv.x * 2, y: py + dv.y * 2 };
      return tileToPixel({           // mirror Blinky across "ahead" point
        x: ahead.x + (ahead.x - bt.x),
        y: ahead.y + (ahead.y - bt.y),
      });
    }
    case 'CLYDE':
      return dist2(ghost.tilePos, player.tilePos) > 64
        ? tileToPixel(player.tilePos)        // far: chase
        : tileToPixel(ghost.scatterTarget);  // close: flee to corner
  }
}
