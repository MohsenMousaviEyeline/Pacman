CYBER-PAC

// sci-fi arcade · browser-native · zero installs to play

A fully reimagined Pac-Man experience wrapped in a neon-drenched cyberpunk skin — built from scratch using modern web technology and running entirely inside your browser.

// 01 · What is this?

The Big Picture

CYBER-PAC is a video game that lives entirely on a web page — no download, no installation, no app store. You open it in a browser and play instantly.


It recreates the classic Pac-Man arcade experience with a futuristic cyberpunk visual style: glowing neon walls, electric particle effects, and four AI-driven ghost enemies that each think differently about how to hunt you.


The game saves your high score locally on your device so it persists across visits, and it supports both keyboard and touchscreen controls.

// 03 · The Technology Stack

What It's Built With

Think of the stack as the layers of a building. Each technology has a specific role, and they all work together to make the game possible.

⚛️
UI Framework
React 19

The engine that manages what you see on screen — the score counter, lives display, and game-over messages. React re-draws only the parts that change, making the interface fast and smooth.

facebook / meta
🔷
Language
TypeScript

A supercharged version of JavaScript (the language all browsers understand natively). TypeScript adds a safety net that catches coding mistakes before the game even runs — like spell-check, but for logic.

microsoft
⚡
Build Tool
Vite

The workshop where the game is assembled. Vite takes all the code files and packages them into a single optimised bundle the browser can load instantly. During development it also refreshes the page in milliseconds after every change.

evan you
🎨
Rendering
HTML5 Canvas

The actual "screen" the game draws on — a blank rectangle inside the web page that code can paint pixel by pixel, 60 times per second. Every wall, dot, ghost, and particle effect is drawn here by the renderer.

web standard
🎮
Game Loop
requestAnimationFrame

A browser built-in timer that ticks in sync with your monitor's refresh rate. It's the heartbeat of the game — every tick, positions are updated and the canvas is redrawn. No external game engine is needed.

web api
💾

localStorage

A tiny built-in storage area every browser provides for websites. CYBER-PAC uses it to remember your high score across sessions — no server, no account, no internet connection required.

// 04 · How it works — one frame at a time

Inside the Game Loop

The game runs 60 times every second. Here's what happens in each of those tiny moments:

01

Read Input

PacmanGame checks whether you're pressing a key or swiping on touch. Your intent is stored and applied on the next opportunity to turn without hitting a wall.

02

Update State (Engine)

The engine.ts file is the pure logic brain. It moves Pac-Man, checks for dot collisions, triggers power-pellet mode, advances ghost timers, and decides whether you've won or lost — all as pure math, no drawing.

03

Ghost AI Decisions (Ghost)

ghost.ts calculates where each ghost wants to go at every intersection. Each of the four ghosts uses a different targeting strategy (see below) to create varied, intelligent pursuit behaviour.

04

Draw Everything (Renderer)

renderer.ts takes the updated state and paints the canvas: neon walls with glow effects, dots, Pac-Man's chomping animation, ghost bodies, and particle effects on collisions. Then React overlays the HUD on top.

05

Schedule Next Frame

The loop schedules itself to run again at the next monitor refresh via requestAnimationFrame. This repeats indefinitely until you close the tab or lose all your lives.

// 05 · Ghost Intelligence

The Four Adversaries

Each ghost has a distinct personality implemented as a targeting algorithm. Together they create coordinated (but not scripted) pressure on the player.

👻
BLINKY · RED

The direct pursuer. Always targets your exact current position. Gets faster as dots are eaten. The most dangerous in the late game.

👻
PINKY · PINK

The ambusher. Targets 4 tiles ahead of where you're facing — tries to cut you off before you arrive.

👻
INKY · CYAN

The flanker. Uses a combination of Blinky's position and yours to calculate a flanking point — unpredictable until you understand the math.

👻
CLYDE · ORANGE

The shy one. Chases you when far away but retreats to his corner when within 8 tiles. Deceptively disruptive.


Ghosts cycle through global modes: Scatter (retreat to corners) → Chase (hunt) → Scatter → Chase, following a fixed timer sequence. Eating a power pellet temporarily flips them to Frightened mode — they turn blue and you can eat them for bonus points (200 → 400 → 800 → 1600).
// 06 · Project files

Codebase Structure

Every file has exactly one responsibility. Here's a plain-English map:

📁 src/
📁 game/ ← pure game logic, no visuals
📄 types.ts — data blueprints (what a Ghost looks like in code)
📄 constants.ts — all tunable numbers (speeds, scores, colours)
📄 maze.ts — the 28×31 grid: walls, dots, teleports
📄 engine.ts — moves everything, checks collisions, tracks score
📄 ghost.ts — ghost AI targeting & mode transitions
📄 renderer.ts — draws all game graphics onto the canvas
📄 PacmanGame.tsx — game loop, keyboard/touch input, HUD layout
📄 App.tsx — root wrapper that loads the game
📄 main.tsx — entry point, mounts React into the HTML page
📁 public/ ← icons & static assets
📄 index.html — the single HTML page the browser loads
📄 vite.config.ts — build tool configuration
📄 package.json — project manifest & dependency list

For Developers: Getting Started

1
Install Node.js A JavaScript runtime required to build and serve the project locally — download from nodejs.org
2
Install dependencies In the project folder run npm install — downloads all required libraries (~30 MB)
3
Start the dev server Run npm run dev — then open http://localhost:5173 in your browser
4
Production build Run npm run build — outputs a self-contained dist/ folder ready to deploy anywhere
