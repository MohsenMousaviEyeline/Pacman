🤔 What Is This?

CYBER-PAC is a browser-based video game inspired by the legendary 1980s arcade classic Pac-Man — but re-imagined with a neon, sci-fi aesthetic. You control a glowing character navigating a maze, eating dots and power pellets while being chased by four AI-controlled ghosts, each with its own personality and behavior.

The whole game runs inside a single web page — no downloads, no plugins, no servers needed after the initial load. Everything from the maze drawing to the ghost brains to the score tracking happens right inside your browser window.

It was built as a learning project to explore modern web technologies and how to build smooth, interactive games using the same tools used to build everyday websites.

React 19
The most popular library for building web interfaces. It powers the heads-up display (score, lives, menus) overlaid on top of the game canvas. Think of it as the "UI layer" that wraps the game.
🔷
TypeScript
A supercharged version of JavaScript that adds "type safety" — it catches mistakes early, like trying to add a number to a word. Every piece of game data (ghost positions, player state) has a strictly defined shape.
🎨
HTML5 Canvas
A browser drawing board that lets code paint pixels directly. All game visuals — the maze, Pac-Man's chomping mouth, glowing ghosts, star particles — are drawn here 60 times per second.
⚡
Vite
The build tool that takes all the source files and bundles them into a fast, optimized website. During development it refreshes your browser instantly when you change a file — no waiting.
🟢
Node.js
The runtime that lets JavaScript run on your computer (not just in a browser). It's used here only during development to run Vite, install packages, and run tests — the final game doesn't need it.
🧪
Vitest
An automated testing framework. It runs small programs that verify the game logic (ghost AI, collision detection) works correctly without having to manually play the game every time.

🚀 How to Run

You'll need Node.js installed on your computer (download it free from nodejs.org if you haven't already). Then open a terminal in the project folder and follow these steps:

  1. 1
    Install dependencies

    Downloads all the tools and libraries the project needs. You only have to do this once (or when dependencies change).

    npm install

  2. 2
    Start the development server

    Launches a local website. Open your browser to http://localhost:5173 and the game is running live.

    npm run dev

  3. 3
    Play!

    Use Arrow keys or W A S D to move. Press Space or Enter to start or pause. Eat all dots to win a level!

Other commands:   npm run build — create an optimized production version  |  npm test — run automated tests  |  npm run lint — check code quality

📁 Key Files & Folders

Here are the most important files — you don't need to understand them all, but knowing where things live helps if you want to explore the code.

File / Folder What it does
src/PacmanGame.tsx Entry point
The heart of the app. Runs the 60fps game loop, reads keyboard input, and lays out the canvas + HUD.
src/game/engine.ts Game Logic
Updates the game state every frame — moves Pac-Man, checks if he hits a wall, detects dot collection and ghost collisions.
src/game/ghost.ts AI
The "brains" of the four ghosts. Each ghost has a unique strategy: one chases directly, one tries to cut you off, one uses a buddy system, one plays it safe.
src/game/renderer.ts Graphics
Takes the game data and paints everything onto the Canvas: the neon maze walls, the chomping mouth animation, glowing ghosts, and floating score popups.
src/game/maze.ts Map data
A 28 × 31 grid of numbers, where each number means something (wall, dot, power pellet, ghost house, teleport tunnel).
src/game/types.ts Data shapes
TypeScript definitions for everything — what a Ghost looks like, what a Player looks like, what the overall GameData object contains.
src/game/constants.ts Tuning
All the numbers that make the game feel right: character speeds, how many points each action is worth, colors for each ghost.
index.html Shell
A minimal HTML file — just a container. Vite and React inject the actual game into it automatically.
package.json Project config
Lists all the project's dependencies (React, Vite, TypeScript, etc.) and the shortcut commands like npm run dev.
